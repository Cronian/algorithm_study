향상된 정렬
1. 병합정렬
 - 이미 정렬되어 있는 데이터를 하나로 합해서 정렬하는 방법
 - 병합을 위해 데이터를 나눠 정렬하고 합침
 - 정렬할 데이터양이 많아지며 성능이 급격히 떨어짐
 - 성능 O(NlogN)
2. 퀵정렬
 - 분할정복 알고리즘
 - 데이턱 적은경우보다 많은경우 알고리즘 성능이 급격히 좋아짐
 - 최선 / 최악의 경우에 성능이 많이 나빠짐 
 - 빠른속도 구현의 용이성 메모리사용량 적음
 - 최선 O(NlogN) / 최악 O(N^2)
3. 힙정렬
 - 운영체제 네트워크 등 시스템 내부에서 가장 많이 사용하는 정렬 알고리즘
 - 트리구조로 구성. 루트노특 가장 큰값을 가짐
 - 우선순위 큐를 이용해 우선순위를 설정하 후 순위에 따라 정렬하는 알고리즘
 - 최선 일반 최악의 경우에 따른 성능차이가 거의 없음
 - 다운힙 반복시 퀵정렬 알고리즘보다 성능이 나빠짐
 - 성능 O(NlogN)
4. 기수정렬
 - 정렬할 데이터의 자릿수를 이용해 데이터를 정렬하는 방법
 - 자릿수로 데이터를 정렬
 - 데이터르 비교하거나 이동하느 횟수가 거의 없음
 - 성능 O(D(N+Q)) < 자릿수에 정렬성능영향을 받음 >
